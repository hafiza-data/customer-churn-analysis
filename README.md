# Customer Churn Analysis
A data analysis project focused on identifying customer churn patterns using SQL and Power BI.
##  About the Project
In this project, I worked on identifying customers who are likely to churn based on their activity, transactions, and subscription details. The main goal was to understand customer behavior and find patterns that indicate risk.

##  Tools Used
- SQL (PostgreSQL)
- Power BI

##  What I Did
- Created multiple tables like customers, subscriptions, transactions, and activity
- Used SQL joins to combine data from different tables
- Applied logic to classify customers into:
  - Active
  - At Risk
  - Churned
- Built an interactive Power BI dashboard to visualize the results

##  Key Observations
- Customers with more failed payments are more likely to churn
- Customers with no recent activity fall into the "At Risk" category
- Subscription type also impacts churn behavior
